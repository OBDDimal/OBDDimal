pub struct DDManager {
    id: usize,
    pub nodes: HashMap<NodeID, DDNode>,
    pub(crate) var2level: Vec<usize>,
    pub(crate) level2nodes: Vec<HashSet<DDNode>>,
    pub(super) ite_c_table: HashMap<(NodeID, NodeID, NodeID), NodeID>,
    pub(super) apply_c_table: HashMap<(ApplyOperation, NodeID, NodeID), NodeID>,
    views: WeakHashSet<Weak<BddView>>,
}
Expand description

Container combining the nodes list, unique tables, information about current variable ordering and computed table.

Fields§

§id: usize

ID to identify the DDManager

§nodes: HashMap<NodeID, DDNode>

Node List

§var2level: Vec<usize>

Variable ordering: var2level[v.0] is the depth of variable v in the tree

§level2nodes: Vec<HashSet<DDNode>>

Unique Table for each variable. Note that the indices into this table are the depths of the variables, not their IDs. The depth of a variable can be determined through var2level!

§ite_c_table: HashMap<(NodeID, NodeID, NodeID), NodeID>

Computed Table for ITE: maps (f,g,h) to ite(f,g,h)

§apply_c_table: HashMap<(ApplyOperation, NodeID, NodeID), NodeID>

Computed Table for Apply: maps (op,u,v) to apply(op,u,v)

§views: WeakHashSet<Weak<BddView>>

Set of Views for BDDs in this manager

pub fn modify_var_nodes( &mut self, relevant_nodes: &HashSet<NodeID>, vars: &HashSet<VarID>, func: &mut NodeModFunction ) -> HashMap<NodeID, NodeID>

Modifies all nodes in the given BDD(s) corresponding to the given variables using the given function. Also corrects the references in nodes above the modified nodes so the resulting BDD(s) are still valid. The “modification” thereby creates new nodes, so the original BDDs can still be used later.

  • relevant_nodes - A set containing all nodes of the BDDs for which this operation is supposed to be run.
  • vars - The variables, for which the nodes should be “modified”.
  • func - The function which should be applied to the nodes corresponding to the given variables.
§Returns

A HashMap which can be used to translate the root nodes of the original BDDs to the root nodes of the new (“modified”) BDDs.

§Notes

May create temporary nodes, consider using purge_retain_multi or purge_retain afterwards.

pub(crate) fn reduce(&mut self, v: NodeID) -> NodeID

Reduces the BDD. This changes Node IDs, the new Node ID of the function passed to the function is returned. All other known Node IDs are invalid after reduce. Note: Reduction does not imply removing nodes not part of the specified function. Other functions may be present in the tree (multiple root nodes, or root above function).

See also “Graph-Based Algorithms for Boolean Function Manipulation” by Bryant (10.1109/TC.1986.1676819)
